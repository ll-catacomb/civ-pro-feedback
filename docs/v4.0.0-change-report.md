# v4.0.0 — Collapse to a single Claude chain

## Why

Across every stored calibration run, the dual OpenAI+Claude pipeline (v2.1.0–v3.2.1) never beat the simple single chain on band accuracy: the best full-chain result on the eight-fixture benchmark was v1.4.0 (4/8 exact, mean distance 0.625), while v3.x dual runs landed around 2/8 exact. The dominant error was central tendency — DS and LP answers pulled toward P/H — and averaging or cross-judging two compressed opinions cannot fix compression. The judging cascade added roughly half the pipeline's cost and most of its complexity for no measured accuracy gain, and made the audit trail harder for human reviewers to follow.

## What changed

- Every stage — both responsiveness passes, issue map, evidence rerank, blind evaluation, coaching draft, skeptical judge, and the post-hoc calibration analysis — now runs on the Claude API (`claude-opus-4-8` by default, `ANTHROPIC_WORK_MODEL` / `ANTHROPIC_JUDGE_MODEL` overrides).
- Removed the dual pipeline, cross-model judges, feedback merge, and the deterministic voting cascade (`dual-pipeline.ts`, `dual-decision.ts`, `claude-chain.ts`, plus their prompts and tests). The band helpers used to render legacy hedged estimates moved into `outcomes.ts`.
- The blind evaluation's provisional band is the run's band recommendation, exactly as in the single-chain versions that calibrated best. The comparative pairwise-anchor banding prompt (v3.4.x) is unchanged.
- The evaluator sweep is Claude-only and its result shape is single-provider (`band` / `distance` / `metrics`).
- OpenAI usage is now limited to optional retrieval embeddings (`text-embedding-3-large`); without `OPENAI_API_KEY`, retrieval degrades to the labeled lexical fallback.
- Claude requests carry the salted pseudonymous identifier as `metadata.user_id`, preserving the privacy posture the OpenAI `safety_identifier` provided.
- Legacy dual runs in `.data/runs.json` keep their `claudeChain` / `crossJudges` / `dualDecision` fields, still validate, and still render in the QA lab, CSV, and JSON exports.

## Compatibility

`PROMPT_VERSION` is `civpro-feedback-v4.0.0`; `CALIBRATION_ANALYSIS_VERSION` is `calibration-analysis-v2.0.0`. No stored data migration is required.
