# Change report: civpro-feedback-v3.1.1

Prepared 2026-07-16. No live calibration run has been made under v3.1.1.

## Evidence

The completed v3.1.0 cohort contained two distinct 2–2 splits under the same chain version:

- 2019 answer — P: OpenAI P / Claude H / Claude judge H / OpenAI judge P; actual P.
- 2014 answer — P: OpenAI P / Claude H / Claude judge H / OpenAI judge P; actual P.

The former rule persisted the higher tied band, producing H for both. Cross-provider confidence had already been removed, but the higher-band rule still resolved every P/H provider deadlock toward the more generous Claude signal.

## Change

A 2–2 split now:

- retains the four-signal fractional hedge;
- remains a manual-review outcome;
- persists the lower tied band as the strict hard estimate;
- does not alter the selected model prompts, anchors, cross-judge prompts, or feedback.

`PROMPT_VERSION` is `civpro-feedback-v3.1.1`, so new runs remain distinguishable from v3.1.0 despite the prompt text being unchanged.

## Counterfactual on v3.1.0

Changing only this deterministic rule would move both split actual-P fixtures from H to P. The eight-run cohort would change from 2/8 exact with mean ordinal distance 0.75 to 4/8 exact with mean distance 0.50. The 2019 DS improvement remains H because both cross-judges agreed on H rather than splitting.
