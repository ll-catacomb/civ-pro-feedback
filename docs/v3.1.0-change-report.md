# Change report: civpro-feedback-v3.1.0

Prepared 2026-07-16. No live calibration run has been made under v3.1.0.

## Scope

This is a narrow pre-run refinement of v3.0.0. The anchor pack, evaluator-owned banding, agreement merge, disagreement cross-judges, and deterministic decision cascade are unchanged.

## Changes

1. **Actual exam points are controlling.** Issue-map criteria now use the examination's exact stated point value, without normalization to 100. A separately scored question or subpart receives its stated points; when only a whole-question allocation exists, subissues remain inside that single criterion. Unallocated criteria use `null`. Cross-cutting skills remain qualitative and receive no invented points.
2. **Two-sided boundary checks return to the evaluator.** `Evaluation` now requires concise `whyNotHigher` and `whyNotLower` fields. They accompany the evaluator's band rather than reviving the separate band-assessment stage. Cross-judges receive them when chains disagree, and the Quality Lab displays both models' boundary checks.
3. **Legacy compatibility.** Old saved evaluations are supplied empty boundary strings at read time so v1–v2.2 archives still validate. Existing numeric issue-map weights remain valid.
4. **Versioning.** `PROMPT_VERSION` is `civpro-feedback-v3.1.0`; input hashes therefore separate this batch from all prior runs.
